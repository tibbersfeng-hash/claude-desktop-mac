## 一、功能概述

### 1.1 背景

Claude Desktop Mac 的核心定位是**替代 Claude Code CLI**，为本地 Claude Code 提供图形化前端界面。Phase 1 是整个项目的技术基石，负责建立 GUI 应用与 Claude Code CLI 之间的通信桥梁。

### 1.2 目标

构建一个稳定、高效、可靠的 CLI 连接层，实现：
- 自动检测和启动 Claude Code CLI
- 建立双向通信管道
- 消息协议的序列化与反序列化
- 流式响应的实时处理
- 错误处理与自动重连

### 1.3 范围

**包含：**
- CLI 环境检测与启动
- 通信管道建立（Socket/stdio/HTTP）
- 消息协议层
- 流式响应处理
- 连接状态管理
- 错误处理与重连机制

**不包含：**
- UI 界面实现（Phase 2）
- 会话管理逻辑
- 工具调用可视化
- 文件操作功能

## 二、用户故事

### US-1: CLI 自动检测

**作为** Claude Desktop 用户
**我希望** 应用能自动检测本机是否安装了 Claude Code CLI
**以便于** 我不需要手动配置路径就能开始使用

**验收标准：**
- 自动检测常见安装路径
- 支持自定义 CLI 路径配置
- 显示检测状态和结果

### US-2: 一键连接

**作为** Claude Desktop 用户
**我希望** 点击一个按钮就能连接到 Claude Code CLI
**以便于** 快速开始对话

**验收标准：**
- 一键启动并连接
- 显示连接进度
- 连接成功后自动进入就绪状态

### US-3: 实时响应流

**作为** Claude Desktop 用户
**我希望** 能看到 Claude 的回复逐字显示
**以便于** 不需要等待完整响应就能开始阅读

**验收标准：**
- 支持流式输出
- 实时显示响应内容
- 正确处理多行内容

### US-4: 断线重连

**作为** Claude Desktop 用户
**我希望** 当连接意外断开时应用能自动重连
**以便于** 我的工作流不被中断

**验收标准：**
- 自动检测连接断开
- 自动尝试重连
- 重连失败时通知用户

### US-5: 错误提示

**作为** Claude Desktop 用户
**我希望** 当出现问题时能看到清晰的错误提示
**以便于** 我能知道如何解决

**验收标准：**
- 错误信息清晰易懂
- 提供解决建议
- 支持错误日志查看

## 六、技术要点

### 6.1 通信方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **stdio** | 简单可靠、无需额外配置 | 双向通信稍复杂 | 推荐（P0） |
| Unix Socket | 高性能、低延迟 | 需要配置路径 | 推荐（P1） |
| HTTP API | 标准化、易于调试 | 需要端口管理 | 备选（P1） |

**建议：** Phase 1 优先实现 stdio 方案，后续迭代增加 Unix Socket 支持。

### 6.2 消息协议设计（草案）

```swift
// 发送消息
struct OutgoingMessage: Codable {
    let type: MessageType
    let content: String
    let sessionId: String?
    let metadata: [String: String]?
}

// 接收消息
struct IncomingMessage: Codable {
    let type: ResponseType
    let content: String?
    let delta: String?        // 流式增量
    let toolCall: ToolCall?   // 工具调用
    let error: ErrorInfo?     // 错误信息
    let isComplete: Bool
}

enum ResponseType: String, Codable {
    case text          // 文本内容
    case delta         // 流式增量
    case toolCall      // 工具调用
    case toolResult    // 工具结果
    case error         // 错误
    case done          // 完成
}
```

### 6.3 连接状态机

```
    ┌──────────┐
    │   Idle   │ ◀─────────────────────┐
    └────┬─────┘                       │
         │ start()                     │
         ▼                             │
    ┌──────────┐                       │
    │Connecting│                       │
    └────┬─────┘                       │
         │ success                     │
         ▼                             │
    ┌──────────┐      disconnect()     │
    │ Connected│ ──────────────────────┤
    └────┬─────┘                       │
         │ error                       │
         ▼                             │
    ┌──────────┐      retry failed     │
    │   Error  │ ──────────────────────┘
    └────┬─────┘
         │ retry success
         │
         └──────────▶ [Connected]
```
